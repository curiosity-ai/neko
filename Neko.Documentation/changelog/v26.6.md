* **Feature**: Changelogs are now built from a **folder** instead of a single hand-maintained file. Drop a folder anywhere in your project, mark it with `changelog: true` in its `index.yml` (plus a `title`/`description`), and add one Markdown file per release named after its version (`v26.6.md`, `1.2.0.md`, …). Neko parses each file name as a version, sorts the entries newest-first, and renders a single timeline page at the folder URL — each entry shows a version badge, an optional headline (frontmatter `title`/`label`) and `date`, followed by that file's body. The folder collapses to one sidebar entry, the version files are not emitted as standalone pages, and the aggregated page is what gets indexed for search and listed in the sitemap. Documented at `configuration/changelog.md`, with a `changelog` skill in the starter template and a `neko-changelog-folder` snippet in `templates.json`. The Neko changelog you are reading now uses this model.
* **Feature**: Added a `quiz` component — a ` ```quiz ` fenced block whose YAML body defines a self-scoring multiple-choice comprehension check. Questions render radio buttons for a single correct `answer` (0-based index) or checkboxes for multiple `answers: [..]`, with an optional per-question `explain`. **Check answers** grades client-side, highlights correct/incorrect options, and reveals explanations; score and answered-state persist per-browser in `localStorage` (no backend). Dispatched from the fenced-block renderer alongside `mermaid`/`force-graph`, documented at `components/quiz.md`, with a `quiz` skill in the starter template and a `neko-quiz` editor snippet.
* **Fix**: `neko gen-images` and `neko gen-dark-images` now skip `[!img-gen]` directives (and `assets/img-gen/*.png` references) that sit inside an HTML comment. After the first generation pass, the original directive is preserved as a `<!-- ... -->` block above the rendered image — re-running the command was matching the commented-out directive again, burning API tokens and producing nested comments / orphan PNGs. The discovery regex is now paired with a comment-span filter so commented directives are detected and ignored; uncommenting a directive (as the documentation describes) is still the way to opt back into regeneration.
